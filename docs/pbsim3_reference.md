## 1. Overview

PBSIM3 simulates long reads (PacBio and ONT) with realistic error profiles.
It offers two modes for error introduction:

- **Quality Score Model (QSHMM)**: inherited from PBSIM2. Models quality
  scores via an HMM, then introduces errors based on the quality scores.
  Does not distinguish error types (sub/ins/del).
- **Error Model (ERRHMM)**: new in PBSIM3. Models the error process directly
  via an HMM that emits error *types* at each position. This is the focus
  of this document.

## 2. FIC-HMM Error Model

### 2.1 HMM structure

The error model uses a **FIC-HMM** (HMM with Factorized Information
Criteria). FIC is a model selection criterion that determines the optimal
number of hidden states from data — unlike standard HMMs which require
the state count to be specified in advance.

Each hidden state has:
- **Initial probability** `ip[state]`
- **Emission distribution** over 4 categories: `ep[state][0..3]`
  - 0 = match (correct base)
  - 1 = substitution
  - 2 = insertion
  - 3 = deletion
- **Transition probabilities** to all states: `tp[state][next_state]`

### 2.2 Accuracy stratification

Separate HMMs are trained per read-accuracy level. The model file contains
independent (IP, EP, TP) parameter sets for each integer accuracy percentage:

```
72 IP 1 2.667e-02       # accuracy 72%, state 1, initial prob
72 IP 2 9.733e-01       # accuracy 72%, state 2, initial prob
72 EP 1 9.985e-01 6.327e-04 8.235e-04 1.548e-08   # state 1: P(match, sub, ins, del)
72 TP 1 4.978e-01 4.826e-01 1.945e-02 ...          # state 1 → state 1,2,3,...
```

Different accuracy levels have different numbers of hidden states (determined
by FIC). For example, accuracy 72% has 5 states while accuracy 73% has 11.

At simulation time, a read's accuracy is drawn from a distribution, and the
corresponding HMM is used. For accuracy levels without enough training data,
the nearest available model is interpolated.

### 2.3 Simulation loop

For each position along the reference:
1. If first position: sample initial state from `ip[accuracy]`
2. Otherwise: transition from current state via `tp[accuracy][state]`
3. Sample error type from `ep[accuracy][state]`:
   - 0 (match): copy reference base
   - 1 (substitution): replace with random different base
   - 2 (insertion): insert random base before current position
   - 3 (deletion): skip current reference position
4. Special: deletion probability is multiplied by `hp_del_bias[hp]` for
   homopolymer runs (controlled by `--hp-del-bias` parameter)

### 2.4 Key properties

- **Sequence-independent**: emission probabilities do NOT depend on the
  reference sequence or preceding bases. The model is purely positional
  via the Markov chain structure.
- **No quality score modeling**: the ERRHMM does not produce quality scores.
  Quality scores are either omitted or generated by a separate model.
- **Position dependence**: emerges implicitly from the HMM's state
  transitions — different stretches of a read may occupy different states,
  creating position-varying error patterns.

## 3. Training

### 3.1 Input data

Alignments between real reads and their reference genomes. The alignment
is converted to a sequence of error types:
- Match → 0
- Substitution → 1
- Insertion → 2
- Deletion → 3

### 3.2 Preprocessing

"INDELs in a homopolymer were randomly rearranged to reduce the bias caused
by the habits of the aligner." This prevents alignment artefacts (e.g.
left-alignment of indels) from biasing the model.

### 3.3 FIC-HMM training

The FIC criterion selects both the number of states and the parameters:

1. Train multiple HMMs with different initial parameters (5 trials per
   accuracy level)
2. Select the model with the maximum (lower bound of) FIC
3. Repeat for each accuracy level independently

Training data is stratified: "for 80% accuracy, training data comprised a
read group with an accuracy of 79.5–80.4%."

## 4. Model file format

Plain text, one parameter per line:

```
{accuracy} IP {state} {probability}
{accuracy} EP {state} {p_match} {p_sub} {p_ins} {p_del}
{accuracy} TP {from_state} {p_to_1} {p_to_2} ... {p_to_N}
```

Available pre-trained models:
- `ERRHMM-RSII.model` — PacBio RS II
- `ERRHMM-SEQUEL.model` — PacBio Sequel
- `ERRHMM-ONT.model` — Oxford Nanopore (standard)
- `ERRHMM-ONT-HQ.model` — Oxford Nanopore (high quality)

## 5. Relevance to skiver profile HMM

### What we adopt from PBSIM3
- The idea of latent error-propensity regimes (hidden states) with
  categorical emissions over error types
- The Markov chain structure that creates position-dependent error patterns

### Where our model differs
- We add **dinucleotide context** conditioning (prev_base × true_base)
- We jointly model **Phred quality scores** as a second emission component
- We use a single model rather than per-accuracy stratification (the HMM
  states themselves capture accuracy variation)
- We train on short-read (Illumina) data via skiver dump, not long-read
  alignments

## 6. Code structure

The entire simulator is a single file: `src/pbsim.cpp` (5,879 lines).

| Symbol | Role |
|--------|------|
| `struct errhmm_t` | HMM parameters: `ip`, `ep`, `tp` arrays indexed by accuracy and state |
| `set_errhmm()` | Loads model file, populates `errhmm` struct |
| `simulate_by_errhmm()` | Main simulation loop for random reads |
| `simulate_by_errhmm_trans()` | Simulation with transcript model |
| `simulate_by_errhmm_templ()` | Simulation with template model |
| `init2state`, `tran2state`, `emis2err` | Lookup tables for fast sampling (CDF → state/emission) |
